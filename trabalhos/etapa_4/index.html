Resumo

Nessa etapa, evoluímos a ideia e implementamos a primeira versão funcional do nosso Sistema de Processamento Visual (SPV) para reconhecimento de placas de veículos brasileiros no padrão Mercosul. O SPV foi desenvolvido utilizando Python e a biblioteca OpenCV, com foco na detecção e leitura de placas em imagens estáticas e vídeo em tempo real.

Tecnologias Utilizadas

O núcleo do sistema foi construído com Python, utilizando as seguintes bibliotecas principais:

  • OpenCV (cv2): Para todas as operações de processamento de imagem, como detecção de bordas, transformações e manipulação de vídeo.
  • Pytesseract e EasyOCR: Motores de Reconhecimento Óptico de Caracteres (OCR) para extrair o texto das placas detectadas. O sistema permite escolher entre eles ou usar ambos em paralelo.
  • NumPy: Para manipulação eficiente de arrays e matrizes, fundamental em processamento de imagem.
  • Multiprocessing: Para executar o OCR em processos separados, evitando que a interface de vídeo em tempo real trave durante a análise.

Implementação Explicada

O pipeline do nosso SPV, implementado no script plates_recognizer.py, segue uma sequência lógica de passos para identificar e ler uma placa:

  1. Entrada e Pré-processamento: O sistema aceita como entrada uma imagem estática ou um fluxo de vídeo da webcam. Cada quadro é redimensionado para uma largura padrão (ex: 640 pixels) para otimizar a performance. Se um arquivo de calibração de câmera for fornecido, a distorção da lente é corrigida.
  2. Detecção de Regiões de Interesse (ROI): Para localizar as placas, utilizamos uma abordagem híbrida:
    • Haar Cascade: Um classificador pré-treinado (haarcascade_russian_plate_number.xml) realiza uma busca rápida por objetos com características de placas.
    • Detecção de Contornos: Em paralelo, o sistema converte a imagem para escala de cinza, aplica um filtro de Canny para detectar bordas e procura por contornos retangulares que correspondam à proporção de uma placa veicular (aproximadamente 4:1).
  3. Filtragem e Seleção de ROIs: As regiões encontradas pelos dois métodos são combinadas, e as duplicatas são removidas para evitar processamento redundante. As ROIs mais promissoras são selecionadas para a próxima etapa.
  4. Reconhecimento Óptico de Caracteres (OCR): Cada ROI selecionada é enviada para um processo de OCR assíncrono. Para melhorar a acurácia, aplicamos diferentes pré-tratamentos na ROI (equalização de histograma, limiarização adaptativa) antes de passá-la para o motor (Tesseract ou EasyOCR).
  5. Pós-processamento e Validação: O texto extraído pelo OCR é "limpo", removendo caracteres não alfanuméricos. Em seguida, ele é validado contra o padrão de placas Mercosul (LLLNLNN). O sistema também realiza substituições comuns (ex: 'O' por '0', 'I' por '1') para corrigir erros frequentes do OCR.
  6. Exibição do Resultado: Em modo de vídeo, a placa validada é exibida na tela, com um retângulo desenhado ao redor da região detectada. Para evitar oscilações, uma placa só é considerada "confirmada" após ser detectada consistentemente em vários quadros consecutivos.

Testes e Validação

Para facilitar a validação do sistema em diferentes condições de iluminação e ângulos, desenvolvemos uma ferramenta auxiliar: um gerador de placas customizadas em HTML. Isso nos permitiu testar o reconhecimento usando a câmera do celular, exibindo placas com textos variados diretamente na tela.
